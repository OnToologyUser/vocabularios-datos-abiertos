3. Description back to ToC

La siguiente figura muestra la clase esaloj:Alojamiento, definida en este vocabulario, así como sus subclases esaloj:Hotel, esaloj:Hostal, esaloj:Albergue, esaloj:Apartahotel, etc., y las relaciones de estas clases con la taxonomía de clases de las ontologías de GeoNames, schema.org y la ontología Cruzar.

Y en la siguiente figura se muestran las propiedades definidas en este vocabulario, esaloj:numHabitaciones y esaloj:numCamas, que permiten describir el número de habitaciones y camas de un alojamiento, y esaloj:categoria, que permite hacer referencia a la categoría del alojamiento del que se trata (número de estrellas, número de llaves, etc.), definido en alguna clasificación SKOS, que no se ha determinado aún. Asimismo hemos incluido en la imagen algunas de las propiedades que son relevantes para alojamientos, y que proceden de otras ontologías:

  • vcard:hasAddress, schema:address y locn:address, para representar la dirección postal del alojamiento.
  • geosparql:hasGeometry, geo:lat y geo:long, para representar las coordenadas geográficas del alojamiento.
  • estur:tipoAccesibilidad, para representar información sobre accesibilidad del alojamiento.

También se pueden utilizar, y se recomienda utilizar, propiedades generales de Dublin Core (por ejemplo, dct:title y dct:identifier) y otras propiedades generales como rdfs:label, rdfs:comment, owl:sameAs, etc.

3.1 Ejemplos de uso back to ToC

A continuación mostramos algunos ejemplos del uso de algunas de estas clases y propiedades, con ejemplos procedentes del portal de datos abiertos de Zaragoza. Utilizaremos en todos estos ejemplos los formatos Turtle y JSON-LDa>

Veamos cómo podemos representar un hotel de Zaragoza (por ejemplo, el Hotel Delicias, cuya URI es http://www.zaragoza.es/api/recurso/turismo/alojamiento/1068). Como se puede ver, declaramos que es una instancia de nuestra clase esadm:ComunidadAutonoma, que tiene como geonames:featureCode una referencia al recurso geonames:S.HTL y como geonames:featureClass una referencia al recurso geonames:S, y que tiene una serie de datos que parecen "repetidos", como su nombre común y nombre oficial. Esta "repetición" de los mismos valores con distintas propiedades es algo que permite facilitar el acceso a nuestros datos a aplicaciones que utilizan unos u otros vocabularios.

           esaloj:numHabitaciones  40;
           esaloj:numCamas         68;
