3. Description back to ToC

En la siguiente figura se muestra cómo este vocabulario extiende la W3C Semantic Sensor Network Ontology. Se crean las siguientes clases:

  • estrf:TrafficIntensityObservation, que aglutina cada una de las observaciones que se ofrecen sobre el tráfico (en este caso, sobre intensidad de tráfico). Esta clase es subclase de una más general, estrf:TrafficObservation, que se puede extender con cualquier otra subclase para medir otros elementos del tráfico en el futuro.
  • estrf:TrafficIntensitySensorOutput y estrf:TrafficIntensityObservationValue, que almacenan los resultados de la observación, es decir, la medición correspondiente sobre intensidad del tráfico.
  • estrf:TrafficIntensitySensor permite identificar el sensor o sistema que ha realizado la medición. En muchas ocasiones no será interesante proporcionar esta información, y por tanto no tiene por qué ofrecerse.
  • estrf:TrafficIntensity es una instancia que representa el tipo de propiedad que se está midiendo.
  • escjr:TramoVia se refiere al elemento que se está midiendo, que será un tramo de vía, según se define en el vocabulario de callejero.
